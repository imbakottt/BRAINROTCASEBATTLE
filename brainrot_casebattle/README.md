# Brainrot Case Battle — Online Edition

Готовый статический сайт с Supabase Realtime. Его можно опубликовать на Vercel, Netlify или Cloudflare Pages.

## Что уже работает

- счётчик игроков онлайн через Realtime Presence;
- живая лента последних дропов;
- общий рейтинг по стоимости аккаунта и победам;
- комнаты PvP: один игрок создаёт, второй входит, результат определяется серверной SQL-функцией;
- одиночный режим, коллекция и прогресс из исходной версии сохранены;
- если Supabase ещё не настроен, сайт открывается в демонстрационном режиме без ошибок.

## 1. Создайте базу

1. Зарегистрируйтесь на https://supabase.com и создайте проект.
2. Откройте **SQL Editor → New query**.
3. Вставьте содержимое файла `supabase.sql` и нажмите **Run**.
4. Откройте **Authentication → Providers → Anonymous** и включите Anonymous Sign-Ins.
5. Откройте **Project Settings → API** и скопируйте:
   - Project URL;
   - anon / public key.

## 2. Подключите сайт

Откройте `config.js` и заполните значения:

```js
window.BRAINROT_CONFIG = {
  supabaseUrl: "https://ВАШ-ПРОЕКТ.supabase.co",
  supabaseAnonKey: "ВАШ_ANON_KEY"
};
```

`anon key` разрешено размещать на сайте: доступ ограничивает RLS из `supabase.sql`. Никогда не вставляйте `service_role key`.

## 3. Проверьте локально

Не открывайте `index.html` двойным кликом. Запустите локальный сервер из папки проекта:

```bash
python3 -m http.server 8080
```

Затем откройте http://localhost:8080. Для проверки PvP откройте сайт в двух разных браузерах или обычном и приватном окне.

## 4. Опубликуйте на Vercel

### Без GitHub

1. Установите Vercel CLI: `npm i -g vercel`.
2. В терминале откройте папку проекта.
3. Выполните `vercel` и подтвердите настройки.
4. Для публикации основной версии выполните `vercel --prod`.

### Через GitHub

1. Создайте репозиторий и загрузите в него все файлы этой папки.
2. На https://vercel.com нажмите **Add New → Project**.
3. Импортируйте репозиторий.
4. Framework Preset: **Other**; Build Command оставьте пустой; Output Directory: `.`.
5. Нажмите **Deploy**.

Netlify: перетащите всю папку в Netlify Drop. Cloudflare Pages: Build command пустой, output directory `/`.

## Важно перед коммерческим запуском

Текущая экономика игры совместима с исходным локальным сохранением. Для проекта с реальными деньгами нужно перенести баланс, инвентарь и открытие кейсов на закрытый сервер, добавить античит, возрастные ограничения, правила и проверку законодательства вашей страны. Не используйте эту сборку для ставок или вывода денег.
